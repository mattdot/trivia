# Question library

The **source of truth** is the set of per-game files in `sets/*.json`. Everything
else here is derived and regenerated by `scripts/build_index.py`:

- `index.jsonl` — one question per line; handy for `grep`/`jq` and fast dedup.
- `INDEX.md` — human-readable roll-up: totals, dimension histogram, every set.

## Set file schema

```json
{
  "set_id": "2026-06-15-us-states",        // unique; usually the filename stem
  "topic": "U.S. states",
  "created": "2026-06-15",
  "masthead": {                            // strings for the rendered HTML guide
    "eyebrow": "TRIVIA FIELD GUIDE",
    "title_html": "Reason Your Way <span class=\"num\">Across</span> the States",
    "title_plain": "Reason Your Way Across the States",
    "dek": "one-line intro",
    "footer_html": "short sourcing note"
  },
  "plan": {                                // the variety audit (see the skill)
    "dimensions": ["latitude", "area", "..."],
    "unique_ratio": 1.0,
    "doubled": []
  },
  "questions": [
    {
      "id": "2026-06-15-us-states-01",
      "cat": "Geography \u00b7 Latitude",   // display eyebrow on the card (a light hint)
      "dimension": "latitude",             // normalized axis — used for variety + dedup
      "archetype": "extreme-north",        // optional ranking shape
      "q": "the question stem",            // 6th-grade wording, no giveaways
      "answer_subject": "Olympia, Washington", // normalized answer — used for dedup
      "state": "Olympia, Washington",      // display answer (HTML template field)
      "detail": "the specific ranked fact",
      "why": "why it's interesting (with a number)",
      "more": "follow-on (runner-up, margin, caveat)",
      "source": "source(s) supporting the claim"
    }
  ]
}
```

### Required per question
`q`, `state`, `dimension`, `answer_subject`. The rest are strongly recommended —
`render.py` uses `cat/q/state/detail/why/more/source`; variety uses `dimension`.

**Uniqueness:** the cross-game dedup fingerprints the **question** (`q`) only —
the same question must never repeat. `answer_subject` is used to enforce a
per-session frequency cap (no answer over ~8% of a run); the **same answer may be
reused in a different question in another set.**

### Why two answer fields?
`state` is what shows on the card (named `state` because the template was first
built for U.S. states; it holds any answer). `answer_subject` is the normalized
form used for the per-session answer-frequency cap, so "Olympia, Washington"
reliably matches itself within a run.
